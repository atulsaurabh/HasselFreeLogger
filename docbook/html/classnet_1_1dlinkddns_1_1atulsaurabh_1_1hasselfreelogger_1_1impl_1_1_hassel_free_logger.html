Detailed Description

Dependencies

This implementation is depending upon Log4j. To use this class in Maven project following dependency need to added in the pom file. 

<dependency>
<groupId>log4j</groupId>
<artifactId>log4j</artifactId>
<version>1.2.17</version>
</dependency>

Otherwise Log4j dependencies need to be satisfied manually.

Purposes

Purpose: of this class is to provide a hassle free logging technique. The implementation supports rolling based logging as well as non rolling based logging mechanism. The rolling option can be activated by setting rollingOn field.

Using this class following level of log can be recorded 

 org.apache.log4j.Level#ALL, org.apache.log4j.Level#DEBUG,
 org.apache.log4j.Level#ERROR, org.apache.log4j.Level#FATAL,
 org.apache.log4j.Level#INFO and org.apache.log4j.Level#WARN

Configuration

On the basis of above mentioned org.apache.log4j.Level and if the rolling is active then different type of log is recorded in different files. These files can be configured by changing inside ResourceBundle file. The following keys are used to configure the log files:

  • log.all : To log org.apache.log4j.Level#ALL
  • log.debug : To log org.apache.log4j.Level#DEBUG
  • log.error : To log org.apache.log4j.Level#ERROR
  • log.fatal : To log org.apache.log4j.Level#FATAL
  • log.info : To log org.apache.log4j.Level#INFO
  • log.warn : To log org.apache.log4j.Level#WARN

Along with the files, the director for log repository can also be configured by setting the key log.directory in ResourceBundle file.

Features

The rolling based logging mechanism provides a way to log the message on the basis of date and time. The date and time option is configurable. The format can be decided using datePattern.

The format of content is also configurable. This format can be modified using conversionPattern
